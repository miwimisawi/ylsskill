# 眼科住院助手 · 前端使用指南

## 启动服务

```powershell
$env:PYTHONUTF8="1"
D:\miniconda\envs\openai\python.exe run_server.py --port 8000
```

启动后终端显示：

```
眼科住院助手启动中...
用户界面:  http://127.0.0.1:8000/
调试界面:  http://127.0.0.1:8000/debug.html
API 文档:  http://127.0.0.1:8000/docs
```

> 第一次收到请求时会自动加载向量索引（约 10–30 秒），之后保持常驻内存。

### 操作说明

**发送问题**
- 点击底部输入框，输入问题，按 `Enter` 发送
- `Shift + Enter` 换行（多行输入）
- 也可直接点击上方的**建议芯片**快速发问

**查询增强选项**（输入框左上角）

| 选项 | 默认 | 说明 |
|------|------|------|
| HyDE | ☑ 开启 | 先生成假设性答案再检索，提升召回率 |
| 多查询 | ☐ 关闭 | 将问题改写为多个角度同时检索 |
| 背景提炼 | ☐ 关闭 | 提取问题的上位概念辅助检索 |

> 三个选项可叠加使用，但每多一项会增加约 5–15 秒耗时。

**阅读回答**
- 回答以 Markdown 格式渲染，支持标题、列表、**加粗**
- 卡片底部显示**来源芯片**，颜色按数据库区分：
  - 橙色：中文教材 / OCR教材
  - 蓝色：BCSC 系列
  - 其他颜色对应不同数据库
- `缓存命中` 蓝色徽章 = 本次回答来自缓存，响应极快

**反馈**
- 👍 有用 / 👎 没用，点击后记录到本地 SQLite，帮助评估质量

**页面导航**
- 右上角 **调试模式** → 跳转到调试界面
- 右上角 **清空对话** → 清除当前会话历史

### 右侧 Pipeline 详情说明

**📋 摘要卡**

| 字段 | 含义 |
|------|------|
| 置信度 | HIGH / MEDIUM / LOW，由 CRAG 最高分决定（阈值 0.35）。HIGH=绿色，MEDIUM=黄色，LOW=红色 |
| 来自缓存 | ✅ 是 = 语义缓存命中，直接返回历史答案；❌ 否 = 全流水线处理 |
| CRAG 最高分 | Rerank 后最高文档的相关性分数（0–1），越高越相关 |
| 检索命中数 | 混合检索（Dense + BM25 + RRF）返回的候选文档总数 |
| Cache ID | 本次回答的唯一 ID，可用于 `/feedback` API |

**⏱ 耗时分析**

每个阶段的耗时以水平条形图展示，长度正比于耗时：

| 阶段 | 典型耗时 | 说明 |
|------|---------|------|
| cache_lookup | < 1s | 语义缓存查询（含嵌入） |
| enhancement | 5–15s | HyDE / 多查询 / 背景提炼（开启才有） |
| retrieval | 5–15s | Dense 检索 + BM25 + RRF 融合 |
| reranking | 1–5s | SiliconFlow Rerank API |
| generation | 10–30s | LLM 生成回答 |
| total | 30–90s | 不含 enhancement 的纯流水线时间 |
| 合计 | 30–100s | 端到端总耗时 |

**🔍 搜索查询**

列出实际发送给检索器的所有查询：
- `原始`：用户原始输入
- `改写`：HyDE 生成的假设性答案（作为查询文本）
- 若开启多查询，还会显示多个角度的改写版本

**📚 引用来源**

Rerank 后按分数排列的前 5 个文档块：
- `#N`：排名
- `[数据库名]`：来源数据库（OCR教材 / 中文教材 / BCSC 等）
- `score=X.XXXX`：Reranker 给出的相关性分数
- `full.md`：原始文件名

> 点击卡片标题栏可折叠 / 展开各卡片。

## 常见问题

**Q：第一次发问很慢（30 秒以上）**  
A：服务启动后第一次加载索引需要时间（加载 ChromaDB + BM25 pickle 文件）。之后的请求只计入流水线本身耗时。

**Q：回答末尾显示"缓存命中"**  
A：近似问题已有缓存（语义相似度 > 0.92），直接返回历史答案，耗时 < 1 秒。若答案有误可点 👎 标记。

**Q：置信度显示 LOW**  
A：CRAG 最高分 < 0.35，说明知识库中相关内容有限，建议结合临床判断。

**Q：服务崩溃 / 无响应**  
A：在终端重新运行启动命令即可。索引已持久化在 `vector_db/` 目录，无需重建。

**Q：如何更新知识库**  
A：将新 / 更新后的 `.md` 文件放入对应数据库目录，然后运行：
```powershell
$env:PYTHONUTF8="1"
D:\miniconda\envs\openai\python.exe debug_indexer.py incremental
```
只重新嵌入有变化的文件，节省 token。
